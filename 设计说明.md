# 中国古代建筑结构可视化 - 设计说明

## 🎨 设计理念与思路

### 一、项目定位

本项目面向**建筑学初学者**和**数字人文爱好者**，通过数据可视化手段降低中国古代建筑结构的学习门槛。设计原则是**"学术性、美观性、易用性"**三者兼顾。

## 三、图表设计思路

### 3.1 雷达图（Radar Chart）

**目的**：直观展示抬梁式与穿斗式在5个维度的综合性能对比。

**设计要点：**
- 指标数量：5个（空间利用率、抗震性、施工难度、造价成本、用材量）
- 雷达形状：圆形（相比多边形更柔和）
- 填充透明度：0.3，避免过度遮挡
- 数据点标记：圆圈，大小适中

**配色方案：**
- 抬梁式：故宫红 #c0392b
- 穿斗式：青花蓝 #3498db

**交互设计：**
- 鼠标悬停显示详细数值
- 图例可点击隐藏/显示对应系列

---

### 3.2 柱状图（Bar Chart）

**目的**：对比不同木材的抗压强度与抗弯强度。

**设计要点：**
- 双柱并列（抗压 vs 抗弯）
- 渐变色填充，增加立体感
- Y轴范围 0-100 MPa，预留余量
- X轴标签清晰，字体加粗

**配色方案：**
- 抗压强度：红色渐变 (#c0392b → #e74c3c)
- 抗弯强度：蓝色渐变 (#3498db → #5dade2)

**交互设计：**
- Tooltip 显示精确数值
- 可切换图例显示/隐藏系列

---

### 3.3 折线图（Line Chart）

**目的**：展示抬梁式与穿斗式在历史上的使用比例变化趋势。

**设计要点：**
- 时间轴：先秦→明清（5个时间节点）
- 平滑曲线：增加视觉美感
- 填充区域：渐变透明度，突出趋势
- 数据点：明显标记

**配色方案：**
- 抬梁式：故宫红 #c0392b
- 穿斗式：青花蓝 #3498db

**交互设计：**
- 鼠标悬停显示朝代及具体百分比
- 区域可点击隐藏/显示

---

### 3.4 散点图（Scatter Chart）

**目的**：分析不同建筑类型与结构形式的适用性关系。

**设计要点：**
- X轴：跨度（米）
- Y轴：高度（米）
- 气泡大小：用材量
- 气泡颜色：结构形式（红/蓝）
- 气泡标签：建筑类型名称

**数据点分类：**
- 宫殿庙宇（抬梁式，大红点）
- 民居（穿斗式，大蓝点）
- 过渡类型（混合结构，可扩展）

**交互设计：**
- Tooltip 显示详细参数和特点说明
- 点击图例可筛选结构类型

## 六、技术实现要点

### 6.1 模块化架构

```
├── index.html          # 主页面（HTML结构）
├── css/
│   └── style.css      # 样式（视觉表现）
├── js/
│   ├── data.js        # 数据层（JSON格式数据）
│   ├── charts.js      # 图表层（ECharts配置）
│   └── main.js        # 控制层（交互逻辑）
```

**设计模式：**
- **MVC 思想**：数据、视图、控制分离
- **可扩展性**：新增图表只需在 charts.js 添加函数
- **可维护性**：数据集中管理，修改一处即可全局更新

### 6.2 性能优化

1. **CDN 加载 ECharts**：利用浏览器缓存
2. **按需初始化**：图表在页面加载完成后才渲染
3. **懒加载（可选）**：大图表可改为滚动到可视区域再加载
4. **无阻塞渲染**：所有 JS 在 DOMContentLoaded 后执行

### 6.3 浏览器兼容性

| 浏览器 | 支持版本 | 测试状态 |
|-------|---------|---------|
| Chrome | 60+ | ✅ 已测试 |
| Firefox | 55+ | ✅ 已测试 |
| Safari | 11+ | ✅ 已测试 |
| Edge | 79+ | ✅ 已测试 |
| IE | - | ❌ 不支持 |

**兼容策略：**
- 使用 ES6 语法，不兼容 IE
- CSS 使用 Flexbox + Grid，现代布局
- 检测 ECharts 加载失败，显示友好提示

## 七、交互设计

### 7.1 用户操作流程

```
打开页面
    ↓
自动初始化4个图表
    ↓
用户浏览图表，鼠标悬停查看详情
    ↓
点击"导出图表" → 选择图表 → 下载 PNG
    ↓
点击"查看原始数据" → 模态框展示 JSON 数据
    ↓
按 ESC 或点击关闭按钮退出模态框
```

### 7.2 微交互设计

- **按钮悬停**：轻微上移 + 阴影加深
- **图表加载**：自动适应容器大小
- **模态框**：半透明遮罩，居中弹出
- **表格行**：悬停高亮，视觉反馈

## 八、内容策略

### 8.1 信息分层

1. **第一层（标题区）**：项目名称 + 副标题
2. **第二层（摘要区）**：项目说明 + 技术栈
3. **第三层（图表区）**：4个核心图表
4. **第四层（数据区）**：详细表格
5. **第五层（总结区）**：特点对比
6. **第六层（参考文献）**：学术依据
7. **第七层（操作区）**：导出/查看数据

### 8.2 文案风格

- **简洁明了**：避免冗长，直接说重点
- **学术语气**：使用专业术语但不过度晦涩
- **引导性**：提示语帮助用户理解图表
- **引用权威**：标注文献来源，增加可信度
